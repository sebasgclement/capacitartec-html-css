Integrador 2 — Mini‑Sitio Accesible con Galería, Modal y Sticky

Construí un mini‑sitio de 2–3 secciones para una marca ficticia (producto, evento, portfolio o blog). El foco está en semántica accesibilidad BEM modal sticky transiciones object‑fit control de texto.

Objetivo: que el sitio sea usable, accesible y prolijo en desktop y mobile, con una UI simple pero cuidada.

Requisitos (técnicos)

  • HTML semántico: usar <header>, <nav>, <main>, <section>, <article>, <footer>.
  • Accesibilidad: alt en imágenes, label + for en inputs, aria-label o aria-expanded donde aplique, focus visible.
  • Metadatos: <title>, <meta name="description">, y favicon.
  • Metodología BEM en clases: .bloque__elemento--modificador.
  • Navegación sticky (position: sticky) o header fijo.
  • Modal para ver detalles (botón “ver más”): abrir/cerrar + aria mínimo.
  • Galería/cards con imágenes usando object-fit: cover y object-position.
  • Transiciones en hover/focus (usar transform/opacity).
  • Control de texto: usar line-clamp (multilínea) o text-overflow: ellipsis (1 línea) en descripciones.
  • Enlaces avanzados: al menos uno con mailto: o tel: + un ancla interna.
  • Details/Summary o FAQ (colapsable) para contenido secundario.
  • Validación visual de formularios (mínima): :focus y :required o aria-required.
  • Sin frameworks. Solo HTML + CSS + un poco de JS vanilla para el modal.

Checklist de entrega

  • ✅ Semántica correcta + estructura lógica.
  • ✅ Accesibilidad mínima (alt, label/for, aria básico, focus visible).
  • ✅ BEM en nombres de clases.
  • ✅ Sticky/fixed sin tapar contenido.
  • ✅ Modal con overlay y cierre por fondo/Escape.
  • ✅ Galería/cards con object-fit y transiciones suaves.
  • ✅ Control de texto (ellipsis o clamp) en descripciones.
  • ✅ Metadatos + favicon + enlaces avanzados.
  • ✅ Código ordenado (indentación, comentarios, archivos organizados).
