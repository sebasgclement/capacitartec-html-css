Especificidad y Herencia en CSS

1. Herencia

La herencia en CSS significa que ciertos estilos aplicados a un elemento padre se transmiten automáticamente a sus elementos hijos. Ejemplo: color de texto, tipografía, tamaño de fuente.

No todas las propiedades se heredan (por ejemplo, margin o border no lo hacen).

Ejemplo:

Este texto es azul porque lo hereda del DIV padre.

Este también es azul y usa la misma fuente.

2. Especificidad

La especificidad determina qué regla CSS tiene prioridad cuando dos o más reglas afectan al mismo elemento.

  • Etiquetas y pseudoelementos: 1 punto
  • Clases, pseudoclases y atributos: 10 puntos
  • ID: 100 puntos
  • Estilo en línea: 1000 puntos
  • !important gana a todo, pero debe usarse solo cuando es realmente necesario.

Ejemplo visual:

Clase (.especial) → rojo

ID (#super-especial) → púrpura

!important → naranja

3. Conflictos y resolución

Cuando múltiples reglas aplican al mismo elemento, se evalúa:

  1. Especificidad (más puntos = más prioridad)
  2. Orden: si la especificidad es igual, gana la última regla escrita.

Ejemplo:

Sin clase ni ID → marrón

Clase (.resaltado) → azul

ID (#unico) → carmesí

Resumen visual

  • Herencia = estilos que se transmiten del padre al hijo automáticamente.
  • Especificidad = puntaje que decide qué estilo gana en caso de conflicto.
  • !important = rompe las reglas, usar solo como último recurso.
  • En empate, gana la última regla definida.
