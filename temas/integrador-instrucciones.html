Integrador 3 — Landing responsive

Demostración final: HTML + CSS (Flex/Grid/Animaciones) + Bootstrap + Accesibilidad.

Abrir demo en nueva pestaña

Objetivo

Construir una landing page profesional con navbar, hero, secciones de contenido, componentes interactivos y formulario de contacto, aplicando todo lo visto en el curso.

Tip Probá la demo arriba para ver el resultado esperado.

Requisitos mínimos

  • Navbar responsive (colapsable en móvil) con enlaces a secciones.
  • Hero con título, texto, typewriter (JS) y CTA que abra un modal.
  • Características (3+ tarjetas) usando grilla de Bootstrap.
  • Galería/Proyectos con CSS Grid (repeat + minmax).
  • Animación on scroll (IntersectionObserver + clase .in).
  • Accordion (FAQ) con componente Bootstrap.
  • Formulario de contacto con controles y validación required.
  • Footer con links y créditos.
  • Accesibilidad: foco visible, alt, y @media (prefers-reduced-motion: reduce).

Consejo Mobile-first + utilitarias de Bootstrap + microanimaciones (transform/opacity).

Pautas técnicas

  • Cargá Bootstrap por CDN (CSS + bundle JS).
  • Usá ../bootstrap/bs-theme.css para la estética general (o tu propio CSS).
  • Semántica: <header>, <nav>, <section>, <footer>.
  • Evitá inline styles salvo casos puntuales.

Estructura sugerida

integrador/
├── index.html
├── css/estilos.css   (opcional)
└── img/              (imágenes locales si usás)

Criterios de evaluación

  • Maquetado y semántica (20%)
  • Responsive (breakpoints + mobile-first) (20%)
  • Bootstrap (grilla, utilitarias, componentes) (20%)
  • Animaciones (hover/scroll/typewriter) (15%)
  • Accesibilidad (alt, foco, reduced-motion) (15%)
  • Calidad de código (orden, legibilidad) (10%)

Extra + Dark mode, offcanvas, scroll-to-top, y mejoras de performance.

Entrega

  • Subí un .zip con tu carpeta integrador/ a Classroom.
  • El entry point debe ser index.html.
  • Probalo en móvil y escritorio antes de entregar.
